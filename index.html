Personal Project · SDLC

GKZipPDF — SDLC Journey

A live production app. Every phase owned, every decision documented. This is what a BA looks like when they build.

Visit Live Product ↗
01

Problem Statement

Identified a real operational friction — extracting PDFs from ZIP files from e-commerce platforms like Nykaa, Myntra. No clean tool existed. The business case was clear.

02

Requirements Gathering & Analysis

Used Interview and Workshop techniques with 2–3 stakeholders. Applied MoSCoW prioritisation: ZIP→PDF & Compress (Must Have), Merge PDF (Should Have), PDF↔Word (Could Have), Image to PDF (Won't Have — Phase 2).

03

Design

Chose a simple, functional layout. Described system architecture and Python Flask tech stack. Prioritised usability over aesthetics.

04

Development

Vibe-coded using LLMs, applying BA thinking to prompt engineering and requirement translation. The spec I wrote became the blueprint for every component.

05

Testing

Unit Testing (each component independently), System Testing (bulk upload of up to 100 ZIP files simultaneously), UAT (team members and friends gave real feedback).

06

Deployment

GitHub for version control → Vercel for hosting (free tier) → Namecheap domain → DNS configuration mapping domain to IP. Fully live at gkzipdf.com.

07

Stakeholder Management

A stakeholder requested Image-to-PDF mid-project. Assessed impact, managed expectations, and proposed a phased approach — core system now, feature in Phase 2. Classic BA move.

Feature Proposal · Swiggy Instamart

Insta-Bundles — Full Proposal

🚀 Swiggy gave me 10-minute delivery — I want to give Swiggy a 10-second checkout. ⚡🛒

User Story: As a fitness-focused "Gym Freak," I want to save my weekly staples as a single "Gym Bundle" — so I don't manually select 15+ favourites every three days.

🎯 The Problem

High-frequency users spend 180–300 seconds manually adding 10+ recurring items every 3 days. High bounce rates during the search-and-add phase.

💡 The Solution

Replace unused "Print" icon with "Bundles" nav entry. Save cart snapshot, name it, repopulate in one tap — reducing Time to Checkout by 85–90%.

📈 Business Impact

Projected 15% increase in monthly order volume per power user. Higher CLV and reduced cart abandonment during search-and-add phase.

🔧 Key Features

  • One-tap bundle population (<500ms)
  • Real-time stock validation
  • Partial success notification if item OOS
  • Max 5 bundles per user profile
  • Cart append, not overwrite

Swiggy Instamart UI — "Bundles" entry point in the bottom navigation bar.
