Personal Project · SDLC

GKZipPDF — SDLC Journey

A live production app. Every phase owned, every decision documented. This is what a BA looks like when they build.

Visit Live Product ↗
01

Problem Statement

Identified a real operational friction — extracting PDFs buried inside ZIP files downloaded from e-commerce platforms like Nykaa, Myntra. No tool existed that handled this cleanly. The business case was clear.

02

Requirements Gathering & Analysis

Used Interview and Workshop techniques with 2-3 internal stakeholders. Applied MoSCoW prioritisation to define scope:

  • Must Have: ZIP → PDF extraction, Compress PDF
  • Should Have: Merge PDF
  • Could Have: PDF ↔ Word conversion
  • Won't Have (Phase 1): Image to PDF
03

Design

Chose a simple, functional layout. Described system architecture and technology stack (Python Flask) to translate into a working UI preview. Prioritised usability over aesthetics.

04

Development

Vibe-coded using LLMs, applying BA thinking to prompt engineering and requirement translation. The spec I wrote became the blueprint for every component.

05

Testing

  • Unit Testing: Each component tested independently
  • System Testing: Bulk upload of up to 100 ZIP files simultaneously
  • UAT: Team members and friends used the product and gave real feedback
06

Deployment

Deployed code to GitHub for version control and feature updates. Chose Vercel for hosting (free tier). Purchased domain on Namecheap and connected via DNS configuration — mapping domain to IP, ensuring accessibility and security.

07

Stakeholder Management

A stakeholder requested an Image-to-PDF feature mid-project. Assessed impact, managed expectations, and proposed a phased approach — core system now, feature in Phase 2. Classic BA move.

Feature Proposal · Swiggy Instamart

Insta-Bundles — Full Proposal

User Story: As a fitness-focused "Gym Freak," I want to save my weekly staples as a single "Gym Bundle" — so I don't manually select 15+ favourites every three days.

🎯 The Problem

High-frequency Instamart users (gym freaks, weekly planners) spend 180–300 seconds manually searching and adding 10+ recurring items every 3 days. High bounce rates during the search-and-add phase.

💡 The Solution

Replace the unused "Print" icon in the bottom nav with a "Bundles" entry point. Users can save a cart snapshot, name it, and repopulate it in one tap — reducing Time to Checkout by 85–90%.

📈 Business Impact

Projected 15% increase in monthly order volume per power user. Higher Customer Lifetime Value and reduced cart abandonment during the search-and-add phase.

🔧 Key Features

  • One-tap bundle population (<500ms)
  • Real-time stock validation
  • Partial success notification if item OOS
  • Max 5 bundles per user profile
  • Cart append, not overwrite

Swiggy Instamart UI — "Bundles" entry point visible in the bottom navigation bar.
